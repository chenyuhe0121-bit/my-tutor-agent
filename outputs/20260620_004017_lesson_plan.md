# 演示模式输出

当前未配置 `LLM_API_KEY`，所以系统没有真正调用大模型。你已经完成了 MVP 链路中的需求组织、联网资料拼接和本地资料拼接。配置 `.env` 后会返回正式内容。

## 你提交给大模型的上下文摘要

```text
请生成一份「单节课教案」。

学生画像：
- 地区：上海
- 年级：新八年级上
- 科目：语文、数学
- 学习阶段：八年级上册预习
- 课程导向：新课标导向
- 家长期望：
  - 授课中老师以身作则
  - 教会孩子正确的学习方法和学习态度
  - 帮助孩子提高自制力
  - 帮助孩子学会做规划
  - 采用鼓励式教育
- 默认教学策略：
  - math:
    - 强调概念理解，不一开始高强度刷题
    - 训练审题、标条件、列关系、规范书写
    - 每节课保留易错点和错题整理要求
  - chinese:
    - 训练课前预习流程：字词、背景、结构、主旨、批注
    - 阅读题强调概括、分析、表达三步
    - 逐步积累作文素材和表达句式
  - habits:
    - 每节课设置一个学习方法小目标
    - 每周设置一个可完成的规划任务
    - 反馈语言具体、温和、可执行

本次生成要求：
- 生成类型：单节课教案
- 本次课科目：语文 + 数学
- 课时长度：120 分钟
- 本次课目标：八年级上语文和数学预习，建立预习方法和规划意识
- 学生最近表现/补充信息：测试生成链路

联网检索资料摘要：
未检索到联网资料，或本次未启用联网搜索。

本地资料摘要：
未提供本地资料。

输出要求：
1. 使用 Markdown 格式。
2. 标题层级清晰。
3. 内容要能直接用于真实授课或沟通。
4. 如果是教案，必须包含课程目标、流程安排、讲解重点、互动问题、练习设计、学习方法训练点、课后任务。
5. 如果是讲义，必须包含知识点梳理、例题、练习、答案解析、错题整理区、自我复盘区。
6. 如果是家长反馈，必须包含本节课内容、孩子表现、优势、问题、后续建议、家庭配合建议，语气专业温和。
7. 如果是学习规划，必须包含周计划、学科安排、自制力训练、复盘机制和家长配合方式。
8. 最后附上“资料使用说明”，说明哪些内容来自联网资料摘要，哪些来自本地资料摘要，不要编造具体来源。

```

## 下一步

1. 在 `.env` 中填写 `LLM_API_KEY`。
2. 如果使用非 OpenAI 模型，修改 `LLM_BASE_URL` 和 `LLM_MODEL`。
3. 重新点击生成，即可获得正式教案、讲义、学习计划或家长反馈。
